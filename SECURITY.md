# Security Policy

## Supported Versions

Security updates are applied to the latest code on the `master` branch.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use one of the following methods:

1. Open a private report via GitHub Security Advisories (preferred).
2. Contact the repository owner directly through GitHub.

When reporting, include:

- A clear description of the issue.
- Steps to reproduce.
- Potential impact.
- Any suggested mitigation.

## Response Timeline

- Initial acknowledgment: within 72 hours.
- Triage and severity assessment: as soon as possible.
- Fix and release timing: depends on severity and complexity.

We will coordinate disclosure responsibly and credit reporters when appropriate.
